# 아트 디렉션 (Art Direction)

> **이 문서의 역할** — clone-tasty-road(메이플스토리 월드, 세로형 모바일 머지 게임)의 모든
> 비주얼 결정의 **기준점**. 새 에셋을 만들거나 고를 때, "이게 우리 게임에 맞나?"를 이 문서로 판정한다.
> 코드/데이터 구현이 아니라 **디자인 판단 기준**을 정의한다.
>
> 확정 결정(2026-06-18):
> 1. **에셋 전략 = 하이브리드** — 몬스터/NPC/캐릭터/배경은 메이플 공식 에셋, 코어 UI·스포너·게임 고유 요소는 오리지널.
> 2. **무드/팔레트 = 메이플 비비드** — 쨍한 원색 + 진한(near-black) 아웃라인, 높은 채도·대비.
> 3. **머지 대상 = 몬스터 진화 사다리** — 공식 메이플 몬스터 스프라이트, 머지 = 진화.
> 4. **세계관 = 풀 리테마** — 음식 테마를 메이플 필드/사냥 세계관으로 전면 교체.

---

## 0. 한 줄 정의

> **"헤네시스 필드에서 알을 부화시켜 몬스터를 모으고 진화시키는, 쨍하고 귀여운 메이플 비비드 톤의 세로형 머지 게임."**

레퍼런스 감성은 Tasty Travels(머지 메커닉/UX) + 메이플스토리(비주얼 IP). 메커닉은 캐주얼 머지, **룩은 100% 메이플**이어야 한다.

---

## 1. North Star — 핵심 7원칙

1. **메이플처럼 보일 것.** 처음 보는 사람이 5초 안에 "메이플 게임"이라고 알아야 한다. 의심되면 공식 메이플 스프라이트 옆에 나란히 놓고 비교한다.
2. **진화가 보일 것.** 티어가 오르면 "더 세졌다 / 자랐다"가 한눈에 읽혀야 한다(크기·화려함·글로우 상승).
3. **보드에서 한눈에 구분될 것.** 7×7 보드에서 인접 티어 두 칸이 **실루엣 + 티어 프레임 색**으로 즉시 구별돼야 한다.
4. **두 소스를 한 톤으로.** 공식 에셋과 오리지널 아트가 섞여도 이질감이 없어야 한다 — 통일 규칙(§2.2)을 항상 통과시킨다.
5. **쨍하지만 눈이 편할 것.** 메이플 비비드는 채도가 높다. 단, 장시간 플레이 가능하도록 **배경은 채도를 한 단계 낮춰** 전경(몬스터/UI)을 띄운다.
6. **세로 모바일 우선.** 모든 구도·여백·터치 타깃은 1080×1920 세로 + 세이프에어리어 기준(§8).
7. **귀여움은 표정에서.** 메이플 몬스터의 매력은 표정이다. 오리지널 요소(스포너 알, 마스코트)도 같은 "큰 눈 + 작은 입" 문법을 따른다.

## 2. 에셋 전략 — 하이브리드 접합 규칙 ⭐

가장 중요한 섹션. 하이브리드의 성패는 **"공식 + 오리지널이 한 게임처럼 보이는가"**에 달려 있다.

### 2.1 무엇을 어디서 조달하나

| 영역 | 조달 | 비고 |
|---|---|---|
| **머지 몬스터(티어 아이템)** | 🟦 공식 | 메이플 몬스터 스프라이트. `msw-search`로 검색. §5 |
| **NPC / 의뢰인 / 캐릭터** | 🟦 공식 | 주문·대화·연출용. 메이플 NPC/아바타 |
| **배경 / 필드 씬 / 환경 오브젝트** | 🟦 공식 | 지역별 필드맵 배경. 단 채도 −1단(§1.5) |
| **플레이어 아바타** | 🟦 공식 | 메이플 아바타 시스템 |
| **티어 프레임 / 뱃지 / 글로우** | 🟧 오리지널 | 몬스터를 보드에 얹는 **통일 래퍼**. §5.2 |
| **스포너(알 / 소환 포탈 / 둥지)** | 🟧 오리지널 | 게임 고유 핵심 요소 |
| **코어 UI**(패널·버튼·헤더·팝업·토스트) | 🟧 오리지널 | §6 |
| **재화 아이콘**(에너지·코인) | 🟧 오리지널 | 이미 제작됨. 메이플 비비드 톤 |
| **UI 세로 배경/프레임** | 🟧 오리지널 | 이미 제작됨(`ui_background_portrait`) |

> 핵심: **머지 대상 = 공식 몬스터**, **그것을 담는 그릇(프레임/UI/스포너) = 오리지널**. 오리지널 아트의 제작 부담이 크게 줄어든다.

### 2.2 두 소스 톤 통일 규칙 (오리지널 아트가 반드시 통과해야 할 6개 게이트)

오리지널 아트를 만들 때, 공식 메이플 스프라이트와 나란히 놓고 아래 6개를 맞춘다.

| # | 게이트 | 기준 |
|---|---|---|
| G1 | **아웃라인** | near-black selout(§4.2). 공식 스프라이트의 외곽선 두께(원본 해상도 1px)와 진하기에 맞춘다. 순수 `#000000` 금지. |
| G2 | **채도/대비** | 메이플 비비드 수준(§4). 전경은 쨍하게, 배경은 한 단계 낮춰. |
| G3 | **광원 방향** | **좌상단 고정.** 모든 오리지널 아트의 하이라이트는 좌상단, 그림자는 우하단. |
| G4 | **접지 그림자** | 보드/필드에 놓이는 모든 개체는 발밑 **타원형 그림자**(어두운 회색 50% alpha, §같은 규격)로 접지감 통일. |
| G5 | **픽셀 밀도** | 오리지널 아트의 픽셀 한 점 크기를 공식 스프라이트와 비슷하게. 너무 매끈하면(고해상) 옆 공식 몬스터와 따로 논다. maple-cartoon 렌더 규격(§8) 사용. |
| G6 | **색온도** | 전체 따뜻한 비비드 레인지 유지. 차가운 회청색 단독 사용 금지(필드 배경 제외). |

### 2.3 공식 몬스터 스프라이트 가져오기 워크플로우

1. `msw-search`(resource search, sprite/animation 카테고리)로 대상 몬스터 검색 → RUID 확보.
2. 보드 아이콘은 **정적 idle 프레임** 사용을 기본으로 한다.
   - 정적 스프라이트가 있으면 그대로.
   - 애니메이션(animationclip)만 있으면 `thumbnail://` 프리픽스로 정지 썸네일 렌더(→ msw-sprite-ruid 스킬).
3. **정규화 규칙**(보드 일관성):
   - **방향 통일** — 모든 몬스터가 같은 방향을 보게(정면 또는 우향, 하나로 고정).
   - **스케일 통일** — 티어별 의도된 크기 램프(§5.3)를 제외하면, 같은 티어 내 몬스터는 프레임 안에서 동일 비율로.
   - **여백 통일** — 128px 소스 기준 사방 패딩 일정하게(실루엣이 프레임에 닿지 않게).
4. `SpriteRUID`가 비면 화면에 **에러 없이 안 보인다**(플랫폼 규칙). 적용 후 반드시 런타임 확인.
5. (선택) 고티어 연출용으로 보드에서 **살아있는 애니메이션 모델**을 쓸 수 있으나, 기본은 정적. 성능·일관성 우선.

## 3. (요약) 머지 = 진화

머지 게임의 핵심 피드백은 "둘을 합치면 더 좋은 게 나온다". 본 게임은 이걸 **몬스터 진화**로 표현한다.

- 티어 상승 = 진화 단계 상승. 플레이어가 이미 아는 메이플 몬스터의 위계를 활용.
- 진화 연출(머지 성공 순간): 빛 번쩍 + 스케일 팝 + 작은 파티클(§9). "성장했다"는 쾌감을 준다.
- 라인업이 생물학적으로 연속일 필요는 없다(머지 관례). **위계감(약→강)**만 명확하면 된다.

## 4. 컬러 팔레트 — 메이플 비비드

### 4.1 코어 브랜드 컬러

| 이름 | HEX | 용도 |
|---|---|---|
| Maple Red | `#E63946` | 주 액션, 위험·강조, 알림 뱃지 |
| Maple Blue | `#2B6CB0` | 보조, 정보, 바다/물 계열 필드 |
| Maple Yellow | `#F6C744` | 하이라이트, 코인/보상 인접, 레전드 |
| Maple Green | `#38A169` | 성공, 신선·자연, 필드 |
| Maple Orange | `#F2802E` | 따뜻한 액센트, 에너지·온기 |

> 순수 원색(`#FF0000`, `#00FF00`) 금지 — 위 비비드 값으로 대체. 쨍하되 살짝 절제된 톤.

### 4.2 아웃라인 (near-black selout) — 메이플 비비드의 핵심

메이플 비비드는 코지 파스텔과 달리 **진한 외곽선**을 쓴다. 단, 순수 검정은 딱딱하므로 **near-black**.

- 공용 아웃라인 베이스: `#1E1A24`(아주 어두운 보라-네이비. 검정처럼 보이되 따뜻함).
- 권장: 각 표면의 **자기 색을 어둡게** 밀되 명도 12~18%까지 내린 near-black 톤. (예: 빨강 표면 → `#3A0F12`에 가까운 매우 어두운 적색).
- 실루엣 대각선엔 **선택적 AA 1px**(외곽선과 채움색 중간색)로 계단 완화. 내부 셰이딩엔 AA 금지(스텝 유지).
- ⛔ 순수 `#000000` 외곽선 금지.

### 4.3 UI 뉴트럴

| 이름 | HEX(목표) | 용도 |
|---|---|---|
| Panel Dark | 기존 `ui_panel_dark` 따름 | 공용 패널/모달 바탕 |
| Cream | `#FFF4E0` | 밝은 패널/카드 바탕 |
| Text Dark | `#2E2622` | 밝은 바탕 위 본문 |
| Text Light | `#FFFBF2` | 어두운 바탕 위 본문 |
| Divider | `#6B5A4E` | 구분선·비활성 외곽 |

### 4.4 시맨틱 / 이코노미

| 의미 | HEX | 비고 |
|---|---|---|
| 에너지(Energy) | `#FFC23C` | 번개 옐로-오렌지. 기존 에너지 아이콘에 맞춤 |
| 코인(Coin) | `#FFD24A` + 브론즈 외곽 | 골드 |
| 보상/경험치 | `#38A169` | 그린 |
| 잠금/비활성 | 그레이스케일 + 채도 −60% | 해금 전 지역·버튼 |
| 위험/실패 | `#E63946` | |

### 4.5 티어 = 프레임 색(희귀도 램프) ⭐

> **중요한 결과:** 공식 몬스터 스프라이트는 **자기 색을 그대로** 가져온다. 따라서 티어를 몬스터 색으로 강제할 수 없다.
> 대신 몬스터를 감싸는 **오리지널 티어 프레임의 색**으로 티어를 인코딩한다. 이 프레임이 동시에 §2.2의 톤 통일 장치 역할도 한다.

| 티어 | 희귀도 | 프레임 색 | 글로우 |
|---|---|---|---|
| T1 | 커먼 | 스틸 `#9AA4B2` | 없음 |
| T2 | 언커먼 | 그린 `#38A169` | 없음 |
| T3 | 레어 | 블루 `#2B6CB0` | 약 |
| T4 | 에픽 | 퍼플 `#8E44AD` | 중 |
| T5 | 레전드 | 골드 `#F6C744` | 강 + 파티클 |

플레이어가 이미 아는 희귀도 색 사다리라 **즉시 학습 불필요**. 보드 가독성과 톤 통일을 한 번에 해결.

## 5. 머지 몬스터 디렉션 (진화 사다리)

### 5.1 라인업 (예시 — 정확한 종은 추후 확정)

`msw-search`로 실제 가용한 몬스터를 확인한 뒤 확정. 현재 데이터의 5티어 + 스포너 구조에 매핑:

| 슬롯 | 예시 | 비고 |
|---|---|---|
| 스포너 | 몬스터 알 / 소환 포탈 / 둥지 | 🟧 오리지널 |
| T1 | 달팽이(Snail) | 🟦 공식 |
| T2 | 파란달팽이(Blue Snail) | 🟦 공식 |
| T3 | 주황버섯(Orange Mushroom) | 🟦 공식 |
| T4 | 뿔버섯(Horny Mushroom) | 🟦 공식 |
| T5 | 슬라임 / 보스급(예: 다크스톤골렘·핑크빈 등) | 🟦 공식, 글로우 강조 |

> 선정 기준: (a) 위계감이 자연스러운가, (b) 실루엣이 서로 충분히 다른가, (c) MSW에서 가용한가.

### 5.2 티어 프레임/뱃지 시스템 (🟧 오리지널, 통일 래퍼)

- **프레임**: 둥근 사각 비비드 프레임. 색 = §4.5 희귀도 램프. near-black 외곽(G1).
- **뱃지**: 프레임 모서리에 티어 숫자(1~5) 소형 뱃지. 색약 사용자 대비(색만으로 구분하지 않게).
- **글로우**: T3↑ 약 → T5 강 + 미세 파티클. 진화 위계의 시각적 상승.
- **접지 그림자**(G4): 프레임 발밑 타원 그림자로 모든 칸 통일.

### 5.3 보드 가독성 규칙 (7×7)

1. **실루엣 우선** — 인접 티어는 몬스터 실루엣 자체가 달라야. 비슷하면 라인업 교체.
2. **티어 프레임 색** — 실루엣이 비슷해도 프레임 색으로 보강.
3. **크기 램프** — 티어 상승 시 프레임 내 몬스터 점유율을 소폭 키워 "커졌다" 신호(과하지 않게, 프레임은 칸에 고정).
4. **칸 채움** — 보드 셀(132 UI단위) 안에서 프레임이 사방 약간 패딩하며 중앙 정렬. 아이콘은 `PreserveSprite=None`으로 칸 중앙 꽉 채움(떠 보이면 `AspectOnly` 의심 — changelog 참조).

### 5.4 표정/귀여움 일관성

- 공식 몬스터는 표정이 이미 있다 — **방향만 통일**(§2.3)하면 마스코트 감성은 자동 확보.
- 오리지널 마스코트(스포너 알 등)는 메이플 문법 따름: **큰 둥근 눈 3~5px + 좌상단 흰 하이라이트 + 작은 입 + (선택)볼터치 핑크 `#F4A8B8`**.

## 6. 스포너 & 코어 UI 디렉션 (🟧 오리지널)

### 6.1 스포너 (알 / 소환 포탈)

- 게임의 시작점이자 정체성 요소. 가장 공들일 오리지널 아트.
- 컨셉: **몬스터 알 / 둥지 / 소환 포탈** 중 택1(추후 확정). 메이플 비비드 + near-black 외곽.
- 상태 표현: 충전 가능(쨍, 살짝 들썩 idle) / 에너지 부족(채도 down, 잠금 느낌).
- 스폰 연출: 알에서 해당 칸으로 "슝" 비행(기존 화로 연출 계승, §9).

### 6.2 UI 컴포넌트 톤

| 컴포넌트 | 디렉션 |
|---|---|
| 패널/모달 | 둥근 모서리, Panel Dark 바탕 + 비비드 액센트 테두리, near-black 외곽 |
| 버튼(주) | Maple Red/Green 채움 + 밝은 텍스트 + 1px 하이라이트(상단)·그림자(하단), 눌림 상태 명확 |
| 버튼(보조) | Cream/뉴트럴 바탕 + Text Dark |
| 헤더 | 코인·에너지 재화 표시. 아이콘 + 숫자, 좌상/우상 고정 |
| 팝업 | 중앙 모달 + 딤(배경 어둡게). 등장 스케일 팝(`BackEaseOut`) |
| 토스트 | 상단/중단 슬라이드-인 후 페이드. 단문, 비차단 |

### 6.3 세로형 레이아웃 원칙

- 기준 해상도 **1080×1920**(세로). 모든 좌표/여백 이 기준.
- **엄지 영역**: 하단 1/3은 조작(스포너·주요 버튼), 상단은 정보(헤더·진행도).
- **세이프에어리어**: 상/하단 노치·홈인디케이터 침범 금지. 핵심 버튼은 안전 영역 안.
- 터치 타깃 최소 크기 확보(작은 아이콘도 충분한 히트 영역).

## 8. 사이즈 & 기술 규격 (MSW)

| 항목 | 값 | 근거 |
|---|---|---|
| UI 기준 해상도 | **1080×1920**(세로) | 프로젝트 표준 |
| 월드 유닛 | 1 unit = 100px | 플랫폼 규칙 |
| 보드 | 7×7, 셀 간격 **132 UI단위**, 스포너 칸 cell(6,3) | merge-board-changelog |
| 머지 아이콘 소스 | **128×128** PNG, maple-cartoon | size-guide |
| maple 렌더 그리드 | 논리 64×64 @ 2px/dot | size-guide |
| 아이콘 칸 정렬 | `PreserveSprite=None`(칸 중앙 꽉) | changelog 학습 |
| 캐릭터/몬스터 일반 | 96×96 ~ 128×128 | size-guide |
| 작은 아이콘/버튼 | 48×48 ~ 64×64 | size-guide |
| 접지 그림자 | 발밑 타원, 어두운 회색 50% alpha | maple-cartoon |
| `SpriteRUID` 빈 값 | 에러 없이 **안 보임** → 적용 후 런타임 확인 | 플랫폼 규칙 |

오리지널 아트 제작 파이프라인: `msw-painter`(maple 스타일) → 128×128 렌더 → 업로드 → RUID. 외곽선은 near-black selout, 광원 좌상단, AA는 실루엣만.

## 9. 모션 / 연출 톤 (Juice)

기존 보드 연출 관례를 계승·확장(merge-board-changelog 참조).

| 이벤트 | 연출 |
|---|---|
| 스폰(알→칸) | 스포너에서 목표 칸으로 비행. 이동 `QuartEaseOut`(~0.32s) + 스케일 0.5→1.0 `BackEaseOut`(톡) |
| 머지 성공(진화) | 빛 번쩍 + 스케일 팝 + 작은 파티클. "성장" 쾌감. 고티어일수록 강하게 |
| 버튼/팝업 | 등장 스케일 팝 `BackEaseOut`, 닫힘 빠른 페이드 |
| 드래그 | 해상도 독립 추적(`ScreenToLocalUIPosition` + grab offset). 손가락에 정확히 붙게 |
| 보상 획득 | 코인/에너지 아이콘이 헤더로 빨려 들어가는 흡입 연출 |

원칙: **모든 핵심 상호작용에 즉각적 시청각 반응**. 단, 과한 연출로 가독성/속도를 해치지 않게.

- [ ] **메이플 룩**: 공식 스프라이트 옆에 놔도 이질감 없는가? (§1.1)
- [ ] **아웃라인**: near-black selout인가? 순수 검정 아닌가? (§4.2/G1)
- [ ] **채도/대비**: 메이플 비비드 레인지인가? 배경은 한 단계 낮췄나? (§4/G2)
- [ ] **광원**: 좌상단 고정인가? (G3)
- [ ] **접지 그림자**: 발밑 타원 그림자 있나? (G4)
- [ ] **픽셀 밀도**: 옆 공식 에셋과 점 크기 비슷한가? (G5)
- [ ] **티어 가독성**(보드): 인접 티어가 실루엣+프레임색으로 구분되나? (§5.3)
- [ ] **진화감**: 티어 오르면 "세졌다"가 읽히나? (§1.2)
- [ ] **세로/세이프에어리어**: 1080×1920, 안전 영역 안인가? (§6.3)
- [ ] **사이즈 규격**: §8 수치 준수했나?
- [ ] **RUID 적용**: 런타임에서 실제로 보이는지 확인했나?

## 11. 금지 사항 (디렉션 파괴)

- ⛔ 순수 `#000000` 외곽선 / 순수 원색(`#FF0000` 등).
- ⛔ 그라데이션·블러·드롭섀도 API로 만든 부드러움(셰이딩은 스텝, 깊이는 수동).
- ⛔ 공식 에셋과 톤(외곽선·채도·픽셀밀도·광원)이 따로 노는 오리지널 아트.
- ⛔ 몬스터마다 제각각인 방향/스케일/여백(보드 일관성 파괴).
- ⛔ 색만으로 티어 구분(색약 대비 — 숫자 뱃지 병행).
- ⛔ 배경이 전경보다 쨍해서 몬스터/UI가 묻히는 구도.
- ⛔ 세이프에어리어 침범, 가로형 구도 가정.
